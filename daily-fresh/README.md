# daily.fresh

Trending IQ quiz built on the daily.dev Public API. Tests how well you've kept up with this week's dev discourse — real titles, real upvotes, real comments. Hackathon entry.

## Identity & leaderboard

No login. Anon identity via a signed cookie (`df_handle`, HttpOnly, SameSite=Lax, HMAC-SHA256).

- First claim of a nickname is free.
- Rename is **one-shot** — after that the handle is locked to the uid forever.
- Leaderboard rows are persisted in SQLite (`./data/daily-fresh.db`), not in cookies.
- Per-uid best-only ranking — flooding the board with low scores is impossible.

## What it does

A pre-warmer pulls the top trending posts from daily.dev every 6h. Each post becomes raw material for one of seven question kinds:

- **Guess the upvotes** — pick the bucket
- **Whose post is this?** — match title to source
- **Pick the tag** — which topic fits best
- **How recent?** — this week, this month, ancient
- **Hot take radar** — comments-heavy or upvote-heavy
- **Summary → title** — match a summary to its post
- **Trivia (LLM)** — optional, generated by a self-hosted model from post metadata

All questions are served from a local SQLite cache; no live daily.dev calls happen on the user request path.

## Stack

- Bun + Elysia, server-rendered HTML
- HTMX + Alpine.js for interactivity
- SQLite (bun:sqlite, WAL mode) for state
- Redis for hot cache
- satori + resvg-js for OG share images
- Docker compose + Caddy for self-host

## Run (one command)

```bash
docker compose up -d --build
```

Builds the app image, starts Redis + app, runs migrations, and pre-warms in the background if the cache is empty (~1 minute first time). Open http://localhost:3737.

Stop with `docker compose down`. SQLite data persists in `./data/`.

## Run (without docker)

```bash
docker compose up -d redis      # Redis only in docker
bun install
bun run migrate
bun run prewarm                 # populates cache (~1 min)
bun run dev                     # serves on $PORT (default 3737)
```

## Scripts

| Command                            | What it does                                              |
| ---------------------------------- | --------------------------------------------------------- |
| `bun run dev`                      | Hot-reload server                                         |
| `bun run start`                    | Production server                                         |
| `bun run migrate`                  | Apply SQLite schema (idempotent)                          |
| `bun run prewarm`                  | Pull from daily.dev and rebuild template questions        |
| `bun run llm-prewarm`              | Generate trivia questions via self-hosted LLM (see below) |

### LLM trivia prewarm

Generates additional trivia-style questions from cached posts using your self-hosted OpenAI-compatible endpoint.

```bash
# default: 2000 questions, qwopus:18b, 2 concurrent batches
bun run llm-prewarm

# custom count + model
bun run llm-prewarm --count=500 --model=qwen3.5:9b

# tune batching
bun run llm-prewarm --batch=8 --concurrency=4

# dry run — generate and print samples without persisting
bun run llm-prewarm --dry
```

Reads `OPENAI_URL`, `OPENAI_TOKEN`, and `LLM_MODEL` from `.env`. Output is merged into the existing trending pool (templates + LLM trivia, mixed at sample time).

## Env vars

Reads `.env` in the project root. Required:

- `DAILY_DEV_PAT` — Personal Access Token from app.daily.dev/settings/api

Optional:

- `PORT` (default 3737)
- `BASE_URL` (default http://localhost:3737)
- `DB_PATH` (default ./data/daily-fresh.db)
- `REDIS_URL` (default redis://localhost:6379)
- `LOG_LEVEL` (debug|info|warn|error, default info)
- `RATE_LIMIT_RPM` (default 50, self-cap below provider's 60)
- `OPENAI_URL`, `OPENAI_TOKEN`, `LLM_MODEL` — required by `llm-prewarm`, otherwise unused
- `COOKIE_SECRET` — required for signed `df_handle` cookie (anon leaderboard identity). Generate via `openssl rand -base64 32`. Rotate in production.
